Quais as principais Funcionalidades?

Funcionalidades do Observable

  • Representação de fluxo de dados: O Observable permite trabalhar com fluxos contínuos de eventos e dados assíncronos ao longo do tempo.
  • Manipulação de sequências: O Observable oferece uma ampla gama de operadores para transformar, filtrar, combinar e manipular sequências de eventos.
  • Composição: Os Observables podem ser facilmente combinados e compostos para criar fluxos de dados mais complexos.
  • Cancelamento: É possível cancelar a assinatura de um Observable, interrompendo a recepção de eventos.
  • Gerenciamento de erros: Os Observables têm recursos embutidos para lidar com erros e tratar exceções durante a transmissão de eventos.
  • Suporte a operações assíncronas complexas: O Observable é adequado para lidar com fluxos de dados assíncronos complexos, como chamadas de API que envolvem várias etapas.

Funcionalidades da Promise

  • Representação de um valor futuro: A Promise representa um valor que pode estar disponível no futuro, seja um resultado bem-sucedido ou um motivo de falha.
  • Tratamento de conclusão: Uma Promise permite que você defina código para ser executado quando a operação assíncrona for concluída com sucesso ou falhar.
  • Encadeamento de ações: É possível encadear várias ações assíncronas em uma sequência lógica usando os métodos then() e catch().
  • Composição simplificada: Promises podem ser facilmente combinadas usando métodos como Promise.all() ou Promise.race().
  • Controle de fluxo: Promises permitem controlar o fluxo assíncrono por meio de instruções condicionais, como if e else, usando a resolução ou rejeição da Promise.
  • Operações assíncronas simples: A Promise é uma escolha adequada para operações assíncronas simples, como carregar um recurso único ou executar uma única tarefa assíncrona.
